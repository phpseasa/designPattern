## 六大设计原则
#### SOLID. 

- s 单一职责. 
- o 开闭原则. 
- l 里氏替换. 
- l 迪米特法则. 
- i 接口隔离.   
- d 依赖倒置.
```
抽象不应该依赖细节, 细节应该依赖抽象,即应针对接口编程,而不是针对实现编程
```

#### 其他原则
- 组合/复用 原则 
- 无环依赖      中介者模式
- 共同封装
- 共同重用
- 好莱坞原则.    依赖倒置
- 高内聚低耦合    last know 最少知道
- 保持界面的简单与傻瓜    KISS
- 不要重复你自己      DRY
- 惯例优于配置
- 契约式设计     依赖倒置,面向接口编程
- 你不需要它     不要一开始把系统设计得非常复杂, 应该简单而且可扩展.
